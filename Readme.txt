## H2
Инициализация git в локальной папке

#### H4
Применяем, чтобы git начал отслеживать историю изменений


1) Открыть терминал
2) Перейти в папку, где хотим использовать ГИТ командой cd
3) Инициализировать гит: git init 

Примечание: мы всегда можем "разгитить" папку командой rm -rf .git

Ура! Репозиторий связан с Гит. Проверяем статус командой git status


#### H4
Основные команды ГИТ при работе с репозиторием

git add --all - добавить файлы для отслеживания 
git add . - добавить всю папку для отслеживания 
git add название файла - добавить конкретний файл для отслеживания

Примечание: git add не коммитит (не сохраняет изменения), а только говорит ГИТ, чтобы он начал следить за этими файлами. Если в данных файлах что-то изменится, ГИТ увидит это и выведет данные через команду git status. Если мы хотим, чтобы изменения сохранились, нужно снова ввести команду git add (далее вариации команды)

git commit - команда для сохранения изменений. 
git commit -m 'это коммит с сообщением, которое мы напишем в кавычках'   

Примечание: посмотреть историю коммитов можно командой git log

## H2
Связывание git в локальной папке с GitHub

1) Создаем репозиторий на GitHub
2) Связываем папку на компьютере через команду git remote add origin далее_ссылка_на_репозиторий_gitHub

Примечание: через команду git remote -v можно увидеть, удалось ли связать папки

3) чтобы запушить данные с локального компьютера на удаленный репозиторий используем команду git push -u origin main

Примечание: для получения прав на push в удаленный репозиторий вам понадобится или shh ключ или токен
